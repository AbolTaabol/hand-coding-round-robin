i'm having a moment where i think

all computation is metaphor

webpages are just files insides of folders on a server

and a server is just a computer that holds folders which don't actually exist instead they just tell the computer how to get to the files which in the end are just bits of data encoded in such a way that the computer can store them in memory which is also a metaphor.

and so...because all computation is metaphor then programming is an abstraction. more so i think it gives us the illusion of control and a feeling of power and i'm wondering...is this why the world wide web, and maybe just computational technology in general, is so messed up? has programming gone so far as to give programmers the illusion that they are actually controlling their computers in such a way that everything they do must be right?

i'm interested in slowing down code and eroding this notion of control. today we are going to make webpages by carefully typing out html and css on each others computers <p>ok with that said...</p>

what is hand coding

all webpages are just files and all of them have a file called the index.html file. this is always the first file that you see when you go to a website. even the really fancy ones.

in this workshop we are going to move away from the bloated websites of 2019 and learn how to build a webpage slowly, by hand, and with care. its possible it will look like a webpage from the 90s but it doesn't have to. the funny thing is that the languages, HTML, CSS, and Javascript are very much the same now as they were back then and its more about your content, how you style it and how you leverage the inherent functionality of these three languages.
